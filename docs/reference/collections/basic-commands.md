# 🧩 Розділ 1: БАЗОВІ КОМАНДИ

## 📌 Навігація та орієнтація
| Команда | Для чого використовується    | Приклад   | Що виведе / зробить     |
| ------- | ---------------------------- | --------- | ----------------------- |
| `pwd`   | показує поточну директорію   | `pwd`     | `/home/user`            |
| `ls`    | список файлів і папок        | `ls`      | file1.txt dir1          |
| `ls -l` | детальний список             | `ls -l`   | права, власник, розмір  |
| `ls -a` | включає приховані файли      | `ls -a`   | `.bashrc .git file.txt` |
| `cd`    | перехід між директоріями     | `cd /etc` | перейде в `/etc`        |
| `cd ..` | перейти на рівень вище       | `cd ..`   | на одну папку назад     |
| `cd ~`  | перейти в домашню директорію | `cd ~`    | `/home/user`            |

## 📌 Додаткові команди для аналізу файлів і структури
### 🔍 stat — детальна інформація про файл
| Команда               | Для чого використовується | Приклад               | Що виведе                         |
| --------------------- | ------------------------- | --------------------- | --------------------------------- |
| `stat file.txt`       | повна інформація про файл | `stat file.txt`       | розмір, inode, права, час доступу |
| `stat -c %s file.txt` | тільки розмір             | `stat -c %s file.txt` | `1024` (байти)                    |
| `stat -c %y file.txt` | час зміни                 | `stat -c %y file.txt` | дата                              |
| `stat -c %U file.txt` | власник                   | `stat -c %U file.txt` | username                          |

📌 Що важливо:

- показує inode
- показує Access / Modify / Change time
- часто використовують у скриптах

### 📄 file — визначає тип файлу
| Команда            | Для чого використовується | Приклад            | Що виведе    |
| ------------------ | ------------------------- | ------------------ | ------------ |
| `file file.txt`    | визначає тип              | `file file.txt`    | ASCII text   |
| `file image.png`   | тип файлу                 | `file image.png`   | PNG image    |
| `file -i file.txt` | MIME-тип                  | `file -i file.txt` | text/plain   |
| `file *`           | перевірка всіх файлів     | `file *`           | список типів |

📌 Дуже корисно:

- коли розширення бреше
- для аналізу бінарників

### 🌳 tree — структура директорій
| Команда     | Для чого використовується | Приклад     | Що виведе       |
| ----------- | ------------------------- | ----------- | --------------- |
| `tree`      | дерево файлів             | `tree`      | структура папок |
| `tree -L 2` | обмеження глибини         | `tree -L 2` | до 2 рівнів     |
| `tree -a`   | з прихованими файлами     | `tree -a`   | включає `.git`  |
| `tree -d`   | тільки папки              | `tree -d`   | без файлів      |
| `tree -h`   | розміри                   | `tree -h`   | з розмірами     |
| `tree /etc` | інша директорія           | `tree /etc` | дерево `/etc`   |

📌 Примітка:
- не завжди встановлений → dnf install tree


### 📊 du — використання диска
| Команда               | Для чого використовується | Приклад               | Що виведе        |
| --------------------- | ------------------------- | --------------------- | ---------------- |
| `du`                  | розмір директорії         | `du`                  | розміри підпапок |
| `du -h`               | у зручному форматі        | `du -h`               | KB, MB           |
| `du -sh`              | загальний розмір          | `du -sh dir`          | `120M dir`       |
| `du -h --max-depth=1` | тільки 1 рівень           | `du -h --max-depth=1` | структура        |

| `df -h` | у зручному форматі        | `df -h` | GB, MB             |

### 🔗 which / whereis — пошук команд
| Команда      | Для чого використовується | Приклад      | Що виведе     |
| ------------ | ------------------------- | ------------ | ------------- |
| `which ls`   | шлях до команди           | `which ls`   | `/usr/bin/ls` |
| `whereis ls` | більше шляхів             | `whereis ls` | bin, man      |

## ✅ Підсумок розділу

у базових командах є:

- навігація (cd, ls, pwd)
- робота з файлами (cp, mv, rm)
- перегляд (cat, less)
- аналіз (stat, file)
- структура (tree)
- диск (du, df)
- допомога (man)
