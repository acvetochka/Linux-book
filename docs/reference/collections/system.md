# 🧰 Розділ 9: СИСТЕМА (systemd, cron, tmux, screen)

## 📌 ⚙️ systemctl — керування службами (systemd)
### 🔹 Основні операції
| Команда                     | Для чого використовується | Приклад                 | Що зробить |
| --------------------------- | ------------------------- | ----------------------- | ---------- |
| `systemctl start service`   | запустити                 | `systemctl start nginx` | старт      |
| `systemctl stop service`    | зупинити                  | команда                 | стоп       |
| `systemctl restart service` | перезапуск                | команда                 | restart    |
| `systemctl reload service`  | перезавантажити конфіг    | команда                 | reload     |
| `systemctl status service`  | статус                    | команда                 | info       |

### 🔹 Автозапуск
| Команда                        | Для чого використовується | Приклад | Що зробить       |
| ------------------------------ | ------------------------- | ------- | ---------------- |
| `systemctl enable service`     | включити автозапуск       | команда | при boot         |
| `systemctl disable service`    | вимкнути                  | команда | off              |
| `systemctl is-enabled service` | перевірка                 | команда | enabled/disabled |

### 🔹 Інформація
| Команда                               | Для чого використовується | Приклад | Що виведе |
| ------------------------------------- | ------------------------- | ------- | --------- |
| `systemctl list-units`                | активні юніти             | команда | список    |
| `systemctl list-units --type=service` | тільки сервіси            | команда | список    |
| `systemctl list-unit-files`           | всі юніти                 | команда | всі       |

### 🔹 Перезавантаження systemd
| Команда                   | Для чого використовується | Приклад | Що зробить |
| ------------------------- | ------------------------- | ------- | ---------- |
| `systemctl daemon-reexec` | перезапуск systemd        | команда | reload     |
| `systemctl daemon-reload` | перечитати конфіг         | команда | оновить    |

## 📌 📜 journalctl — системні логи
| Команда                    | Для чого використовується | Приклад               | Що виведе    |
| -------------------------- | ------------------------- | --------------------- | ------------ |
| `journalctl`               | всі логи                  | команда               | журнал       |
| `journalctl -u service`    | логи сервісу              | `journalctl -u nginx` | тільки nginx |
| `journalctl -f`            | live лог                  | команда               | як tail      |
| `journalctl -xe`           | помилки                   | команда               | debug        |
| `journalctl --since today` | за сьогодні               | команда               | лог          |

## 📌 ⏰ cron — планувальник задач
### 🔹 Основи
| Команда      | Для чого використовується | Приклад | Що зробить |
| ------------ | ------------------------- | ------- | ---------- |
| `crontab -e` | редагувати                | команда | відкриє    |
| `crontab -l` | список                    | команда | покаже     |
| `crontab -r` | видалити                  | команда | очистить   |

### 🔹 Формат cron
```bash
* * * * * command
│ │ │ │ │
│ │ │ │ └─ день тижня (0-7)
│ │ │ └── місяць
│ │ └──── день
│ └────── година
└──────── хвилина
```

### 🔹 Приклади
| Задача        | Запис                 |
| ------------- | --------------------- |
| Кожну хвилину | `* * * * * command`   |
| Щодня о 2:00  | `0 2 * * * command`   |
| Кожні 5 хв    | `*/5 * * * * command` |

## 📌 🖥️ tmux — мультиплексор терміналу

👉 дозволяє:
- кілька сесій
- не втрачати процеси при відключенні

### 🔹 Основи
| Команда                     | Для чого використовується | Приклад | Що зробить |
| --------------------------- | ------------------------- | ------- | ---------- |
| `tmux`                      | нова сесія                | `tmux`  | старт      |
| `tmux new -s name`          | іменована                 | команда | session    |
| `tmux ls`                   | список                    | команда | sessions   |
| `tmux attach`               | підключитись              | команда | attach     |
| `tmux attach -t name`       | конкретна                 | команда | attach     |
| `tmux kill-session -t name` | видалити                  | команда | kill       |

### 🔹 Гарячі клавіші (tmux)
| Комбінація | Що робить            |
| ---------- | -------------------- |
| `Ctrl+b d` | detach               |
| `Ctrl+b c` | нове вікно           |
| `Ctrl+b %` | вертикальний split   |
| `Ctrl+b "` | горизонтальний split |

## 📌 🖥️ screen — альтернатива tmux
### 🔹 Основи
| Команда          | Для чого використовується | Приклад | Що зробить |
| ---------------- | ------------------------- | ------- | ---------- |
| `screen`         | нова сесія                | команда | старт      |
| `screen -S name` | іменована                 | команда | session    |
| `screen -ls`     | список                    | команда | sessions   |
| `screen -r`      | підключитись              | команда | attach     |
| `screen -r name` | конкретна                 | команда | attach     |

### 🔹 Гарячі клавіші (screen)
| Комбінація | Що робить  |
| ---------- | ---------- |
| `Ctrl+a d` | detach     |
| `Ctrl+a c` | нове вікно |

## 📌 🔥 Типові сценарії
| Задача                         | Команда                   |
| ------------------------------ | ------------------------- |
| Перезапустити сервіс           | `systemctl restart nginx` |
| Подивитись лог                 | `journalctl -u nginx`     |
| Додати cron                    | `crontab -e`              |
| Запустити процес і не втратити | `tmux`                    |
| Повернутись до сесії           | `tmux attach`             |

## 📌 🧠 Важливо

👉 systemd = init система  
👉 journalctl = централізовані логи

👉 cron:
- НЕ підходить для складних залежностей → тоді systemd timers

👉 tmux/screen:
- критично для серверів (SSH)

## ✅ Підсумок
- служби → systemctl
- логи → journalctl
- задачі → cron
- сесії → tmux, screen
