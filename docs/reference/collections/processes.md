# ⚙️ Розділ 8: ПРОЦЕСИ

## 📌 📊 Перегляд процесів
### 🔹 ps — знімок процесів
| Команда                | Для чого використовується | Приклад     | Що виведе     |
| ---------------------- | ------------------------- | ----------- | ------------- |
| `ps`                   | процеси поточного shell   | `ps`        | список        |
| `ps aux`               | всі процеси               | `ps aux`    | повний список |
| `ps -ef`               | альтернативний формат     | `ps -ef`    | всі процеси   |
| `ps aux \| grep nginx` | пошук процесу             | команда     | знайде        |
| `ps -u user`           | процеси користувача       | `ps -u bob` | список        |

📌 Важливі колонки:

- PID
- USER
- %CPU / %MEM
- COMMAND

### 🔹 top — моніторинг у реальному часі
| Команда       | Для чого використовується | Приклад | Що робить   |
| ------------- | ------------------------- | ------- | ----------- |
| `top`         | live процеси              | `top`   | оновлюється |
| `top -u user` | по користувачу            | команда | фільтр      |
| `top -p PID`  | конкретний процес         | команда | відстеження |

📌 Клавіші в top:

- k → kill
- q → вихід
- M → сортування по RAM
- P → по CPU

### 🔹 htop — покращений top
| Команда | Для чого використовується | Приклад | Що робить    |
| ------- | ------------------------- | ------- | ------------ |
| `htop`  | зручний UI                | `htop`  | інтерактивно |

📌 Може не бути встановлений:
```bash
dnf install htop
```

## 📌 🔍 Пошук процесів
| Команда         | Для чого використовується | Приклад       | Що виведе  |
| --------------- | ------------------------- | ------------- | ---------- |
| `pgrep name`    | знайти PID                | `pgrep nginx` | PID        |
| `pgrep -l name` | з іменем                  | команда       | PID + name |
| `pidof name`    | PID процесу               | `pidof nginx` | PID        |

## 📌 ❌ Завершення процесів
### 🔹 kill
| Команда       | Для чого використовується | Приклад     | Що зробить |
| ------------- | ------------------------- | ----------- | ---------- |
| `kill PID`    | завершити процес          | `kill 1234` | SIGTERM    |
| `kill -9 PID` | примусово                 | команда     | SIGKILL    |
| `kill -l`     | список сигналів           | команда     | сигнали    |


📌 Основні сигнали:
- 15 (TERM) — м’яке завершення
- 9 (KILL) — жорстке

### 🔹 killall
| Команда           | Для чого використовується | Приклад         | Що зробить  |
| ----------------- | ------------------------- | --------------- | ----------- |
| `killall name`    | по імені                  | `killall nginx` | всі процеси |
| `killall -9 name` | примусово                 | команда         | kill        |

### 🔹 pkill
| Команда         | Для чого використовується | Приклад       | Що зробить |
| --------------- | ------------------------- | ------------- | ---------- |
| `pkill name`    | як grep+kill              | `pkill nginx` | завершить  |
| `pkill -u user` | по користувачу            | команда       | kill       |

## 📌 ⚖️ Пріоритет процесів
### 🔹 nice
| Команда              | Для чого використовується | Приклад | Що зробить     |
| -------------------- | ------------------------- | ------- | -------------- |
| `nice -n 10 command` | задати пріоритет          | команда | lower priority |

### 🔹 renice
| Команда         | Для чого використовується | Приклад | Що зробить |
| --------------- | ------------------------- | ------- | ---------- |
| `renice 10 PID` | змінити пріоритет         | команда | змінить    |


📌 Значення:

- `-20` → найвищий пріоритет
- `19` → найнижчий

## 📌 🔄 Фонові процеси (jobs)
| Команда     | Для чого використовується | Приклад       | Що зробить |
| ----------- | ------------------------- | ------------- | ---------- |
| `command &` | запустити у фоні          | `sleep 100 &` | background |
| `jobs`      | список                    | `jobs`        | jobs       |
| `fg`        | у foreground              | `fg %1`       | поверне    |
| `bg`        | у фон                     | `bg %1`       | background |
| `kill %1`   | kill job                  | команда       | завершить  |

## 📌 🔗 Корисні комбінації
| Команда                  | Для чого використовується | Приклад                     | Що зробить |
| ------------------------ | ------------------------- | --------------------------- | ---------- |
| `ps aux \| grep process` | знайти процес             | команда                     | PID        |
| `pgrep \| xargs kill`    | масове завершення         | `pgrep nginx \| xargs kill` | kill       |
| `top \| grep`            | фільтр                    | команда                     | пошук      |

## 📌 🔥 Типові сценарії
| Задача           | Команда                |
| ---------------- | ---------------------- |
| Знайти процес    | `ps aux \| grep nginx` |
| Дізнатись PID    | `pgrep nginx`          |
| Завершити процес | `kill PID`             |
| Примусово        | `kill -9 PID`          |
| Запустити у фоні | `command &`            |

## 📌 🧠 Важливо

👉 процес = програма в пам’яті  
👉 PID — унікальний ID  

👉 SIGTERM vs SIGKILL:
- TERM → дає шанс завершитись нормально
- KILL → одразу "вбиває"

## ✅ Підсумок
- перегляд → ps, top
- пошук → pgrep
- завершення → kill, pkill
- пріоритет → nice, renice
- фон → jobs, bg, fg
