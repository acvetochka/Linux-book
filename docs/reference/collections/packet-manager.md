# 📦 Розділ 6: МЕНЕДЖЕР ПАКЕТІВ (dnf)

## 📌 📥 Встановлення, видалення, оновлення
| Команда                 | Для чого використовується | Приклад                    | Що зробить     |
| ----------------------- | ------------------------- | -------------------------- | -------------- |
| `dnf install package`   | встановити пакет          | `sudo dnf install nginx`   | встановить     |
| `dnf remove package`    | видалити пакет            | `sudo dnf remove nginx`    | видалить       |
| `dnf reinstall package` | перевстановити            | `sudo dnf reinstall nginx` | заново         |
| `dnf update`            | оновити систему           | `sudo dnf update`          | всі пакети     |
| `dnf upgrade`           | те саме що update         | `sudo dnf upgrade`         | оновить        |
| `dnf downgrade package` | відкотити версію          | `sudo dnf downgrade nginx` | старіша версія |

## 📌 🔎 Пошук пакетів
| Команда              | Для чого використовується | Приклад              | Що виведе    |

## ✅ Підсумок по dnf
- dnf install/remove → керування пакетами
- dnf search/info → інформація
- dnf provides → який пакет містить файл 🔥
- dnf info → з якого repo пакет 🔥


## `rpm` — робота з пакетами напряму

👉 rpm працює без репозиторіїв  
👉 використовується для: 
- локальних .rpm файлів
- діагностики
- перевірки пакетів

### 📌 📥 Встановлення / видалення
| Команда                | Для чого використовується  | Приклад              | Що зробить |
| ---------------------- | -------------------------- | -------------------- | ---------- |
| `rpm -ivh package.rpm` | встановити пакет           | `rpm -ivh nginx.rpm` | встановить |
| `rpm -Uvh package.rpm` | оновити або встановити     | команда              | upgrade    |
| `rpm -e package`       | видалити                   | `rpm -e nginx`       | видалить   |
| `rpm -e --nodeps`      | видалити без залежностей ⚠️ | команда              | небезпечно |

### 📌 Ключі:

- `-i` → install
- `-U` → upgrade
- `-v` → verbose
- `-h` → progress bar

### 📌 🔎 Інформація про встановлені пакети
| Команда           | Для чого використовується | Приклад         | Що виведе |
| ----------------- | ------------------------- | --------------- | --------- |
| `rpm -qa`         | всі пакети                | `rpm -qa`       | список    |
| `rpm -q package`  | перевірити                | `rpm -q nginx`  | версія    |
| `rpm -qi package` | деталі                    | `rpm -qi nginx` | опис      |
| `rpm -ql package` | список файлів             | `rpm -ql nginx` | файли     |
| `rpm -qc package` | конфіги                   | `rpm -qc nginx` | config    |
| `rpm -qd package` | документація              | `rpm -qd nginx` | docs      |

### 📌 🔍 Який пакет містить файл
| Команда                 | Для чого використовується | Приклад                  | Що виведе |
| ----------------------- | ------------------------- | ------------------------ | --------- |
| `rpm -qf /path/to/file` | знайти пакет              | `rpm -qf /usr/bin/nginx` | nginx     |
| `rpm -qf $(which ls)`   | через команду             | команда                  | пакет     |

📌 Важливо:
- працює тільки для встановлених пакетів

### 📌 📦 Робота з `.rpm` файлами (без встановлення)
| Команда                | Для чого використовується | Приклад | Що виведе |
| ---------------------- | ------------------------- | ------- | --------- |
| `rpm -qpi package.rpm` | інформація                | команда | опис      |
| `rpm -qpl package.rpm` | список файлів             | команда | файли     |
| `rpm -qpR package.rpm` | залежності                | команда | requires  |

### 📌 🔐 Перевірка пакетів
| Команда          | Для чого використовується | Приклад        | Що зробить    |
| ---------------- | ------------------------- | -------------- | ------------- |
| `rpm -V package` | перевірка змін            | `rpm -V nginx` | перевірить    |
| `rpm -Va`        | всі пакети                | команда        | audit системи |

📌 перевіряє:
- зміни файлів
- права
- checksum

## 📌 🔥 Типові сценарії
| Задача                   | Команда                |
| ------------------------ | ---------------------- |
| Встановити локальний rpm | `rpm -ivh file.rpm`    |
| Дізнатись пакет файлу    | `rpm -qf /usr/bin/top` |
| Перевірити пакет         | `rpm -V nginx`         |
| Подивитись файли пакета  | `rpm -ql nginx`        |


## 📌 ⚠️ Важливі відмінності rpm vs dnf
| `rpm`                   | `dnf`                |
| ----------------------- | -------------------- |
| локальний               | працює з repo        |
| НЕ вирішує залежності ❌ | вирішує залежності ✅ |
| швидкий                 | зручний              |
| низький рівень          | високий рівень       |


## 📌 🧠 Запам’ятати

👉 rpm:

- не працює з repo
- не тягне залежності
- використовується для:
- діагностики
- локальних пакетів

👉 dnf:

- поверх rpm
- вирішує залежності
- стандарт для роботи

## ✅ Підсумок
- dnf → повсякденна робота
- rpm → низький рівень / debug
