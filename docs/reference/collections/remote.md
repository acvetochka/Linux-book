# 🔐 Розділ 11: ВІДДАЛЕНИЙ ДОСТУП

## 📌 🔑 ssh — підключення до віддаленої машини
### 🔹 Основне використання
| Команда                 | Для чого використовується | Приклад                 | Що зробить   |
| ----------------------- | ------------------------- | ----------------------- | ------------ |
| `ssh user@host`         | підключення               | `ssh user@192.168.1.10` | login        |
| `ssh -p 2222 user@host` | інший порт                | команда                 | порт         |
| `ssh host`              | якщо є config             | команда                 | підключиться |

### 🔹 Виконання команд віддалено
| Команда                   | Для чого використовується | Приклад | Що зробить |
| ------------------------- | ------------------------- | ------- | ---------- |
| `ssh user@host "command"` | виконати команду          | команда | виконає    |
| `ssh host uptime`         | швидка перевірка          | команда | uptime     |

## 📌 🔐 SSH ключі
### 🔹 Генерація ключів
| Команда                 | Для чого використовується | Приклад             | Що зробить    |
| ----------------------- | ------------------------- | ------------------- | ------------- |
| `ssh-keygen`            | створити ключ             | `ssh-keygen -t rsa` | ключі         |
| `ssh-keygen -t ed25519` | сучасний ключ             | команда             | рекомендовано |

### 🔹 Копіювання ключа
| Команда                 | Для чого використовується | Приклад | Що зробить  |
| ----------------------- | ------------------------- | ------- | ----------- |
| `ssh-copy-id user@host` | копіювати ключ            | команда | авторизація |
| `cat ~/.ssh/id.pub`     | показ ключа               | команда | вручну      |

### 🔹 Конфігурація

📄 `~/.ssh/config`
```bash
Host myserver
  HostName 192.168.1.10
  User user
  Port 22
```
👉 після цього:
```bash
ssh myserver
```

## 📌 📂 scp — копіювання файлів
| Команда                      | Для чого використовується | Приклад | Що зробить  |
| ---------------------------- | ------------------------- | ------- | ----------- |
| `scp file user@host:/path`   | upload                    | команда | передасть   |
| `scp user@host:/file .`      | download                  | команда | завантажить |
| `scp -r dir user@host:/path` | папка                     | команда | рекурсивно  |
| `scp -P 2222`                | порт                      | команда | інший порт  |

## 📌 🔄 rsync — синхронізація (кращий варіант)
| Команда                          | Для чого використовується | Приклад | Що зробить |
| -------------------------------- | ------------------------- | ------- | ---------- |
| `rsync -av dir/ user@host:/path` | копіювання                | команда | sync       |
| `rsync -av --delete`             | синхронізація             | команда | mirror     |
| `rsync -z`                       | стиснення                 | команда | швидше     |
| `rsync -e ssh`                   | через SSH                 | команда | secure     |

📌 Переваги:

- копіює тільки зміни
- швидше за scp

## 📌 🔐 SSH тунелі (port forwarding)
### 🔹 Local forwarding
| Команда                         | Для чого використовується | Приклад | Що зробить       |
| ------------------------------- | ------------------------- | ------- | ---------------- |
| `ssh -L 8080:localhost:80 host` | проброс порту             | команда | доступ до remote |


👉 відкриє:
```bash
localhost:8080 → remote:80
```

### 🔹 Remote forwarding
| Команда                         | Для чого використовується | Приклад | Що зробить |
| ------------------------------- | ------------------------- | ------- | ---------- |
| `ssh -R 8080:localhost:80 host` | зворотний тунель          | команда | reverse    |

### 🔹 Dynamic (SOCKS proxy)
| Команда            | Для чого використовується | Приклад | Що зробить |
| ------------------ | ------------------------- | ------- | ---------- |
| `ssh -D 1080 host` | proxy                     | команда | SOCKS      |

## 📌 🔍 Додаткові команди
| Команда      | Для чого використовується | Приклад | Що зробить |
| ------------ | ------------------------- | ------- | ---------- |
| `ssh -v`     | debug                     | команда | verbose    |
| `ssh -i key` | ключ                      | команда | auth       |
| `ssh-agent`  | кеш ключів                | команда | агент      |
| `ssh-add`    | додати ключ               | команда | load       |

## 📌 🔥 Типові сценарії
| Задача                  | Команда                          |
| ----------------------- | -------------------------------- |
| Підключитись до сервера | `ssh user@host`                  |
| Копіювати файл          | `scp file user@host:/path`       |
| Синхронізувати          | `rsync -av dir/ user@host:/path` |
| Налаштувати ключі       | `ssh-copy-id user@host`          |
| Проброс порту           | `ssh -L 8080:localhost:80 host`  |

## 📌 🧠 Важливо

👉 scp vs rsync:
- scp → просто
- rsync → ефективно 🔥

👉 ключі краще ніж пароль:
- безпечніше
- зручніше

👉 SSH config:
- економить час

## ✅ Підсумок
- доступ → ssh
- копіювання → scp
- синхронізація → rsync
- безпека → ключі
- тунелі → -L, -R, -D
