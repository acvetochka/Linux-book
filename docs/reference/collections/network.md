## 📌 🌐 ip — основна утиліта (замість ifconfig)
### 🔹 Адреси та інтерфейси
| Команда             | Для чого використовується | Приклад | Що виведе |
| ------------------- | ------------------------- | ------- | --------- |
| `ip a`              | всі інтерфейси            | `ip a`  | IP адреси |
| `ip addr show`      | те саме                   | команда | список    |
| `ip addr show eth0` | конкретний інтерфейс      | команда | info      |
| `ip link show`      | стан інтерфейсів          | команда | UP/DOWN   |

### 🔹 Керування інтерфейсами
| Команда                                | Для чого використовується | Приклад | Що зробить |
| -------------------------------------- | ------------------------- | ------- | ---------- |
| `ip link set eth0 up`                  | включити                  | команда | UP         |
| `ip link set eth0 down`                | вимкнути                  | команда | DOWN       |
| `ip addr add 192.168.1.10/24 dev eth0` | додати IP                 | команда | IP         |
| `ip addr del ...`                      | видалити IP               | команда | remove     |

### 🔹 Маршрути
| Команда        | Для чого використовується | Приклад | Що виведе |
| -------------- | ------------------------- | ------- | --------- |
| `ip route`     | таблиця маршрутів         | команда | routes    |
| `ip route add` | додати маршрут            | команда | додасть   |
| `ip route del` | видалити                  | команда | delete    |

## nmcli — NetworkManager CLI

👉 використовується для:
- керування мережевими підключеннями
- Wi-Fi / Ethernet
- IP, DNS
- заміна GUI

### 📌 📡 Загальна інформація
| Команда                | Для чого використовується | Приклад | Що виведе   |
| ---------------------- | ------------------------- | ------- | ----------- |
| `nmcli`                | загальний статус          | `nmcli` | overview    |
| `nmcli general status` | стан мережі               | команда | connected   |
| `nmcli device status`  | список пристроїв          | команда | eth0, wlan0 |

### 📌 🔌 Інтерфейси (devices)
| Команда                        | Для чого використовується | Приклад | Що зробить |
| ------------------------------ | ------------------------- | ------- | ---------- |
| `nmcli device show`            | деталі                    | команда | info       |
| `nmcli device disconnect eth0` | відключити                | команда | down       |
| `nmcli device connect eth0`    | підключити                | команда | up         |

### 📌 🌐 Підключення (connections)
| Команда                        | Для чого використовується | Приклад | Що зробить |
| ------------------------------ | ------------------------- | ------- | ---------- |
| `nmcli connection show`        | список підключень         | команда | список     |
| `nmcli connection up name`     | активувати                | команда | connect    |
| `nmcli connection down name`   | вимкнути                  | команда | disconnect |
| `nmcli connection delete name` | видалити                  | команда | remove     |

### 📌 📶 Wi-Fi
| Команда                                        | Для чого використовується | Приклад | Що зробить |
| ---------------------------------------------- | ------------------------- | ------- | ---------- |
| `nmcli device wifi list`                       | список мереж              | команда | SSID       |
| `nmcli device wifi connect SSID password pass` | підключитись              | команда | Wi-Fi      |
| `nmcli radio wifi off`                         | вимкнути Wi-Fi            | команда | off        |
| `nmcli radio wifi on`                          | включити                  | команда | on         |

### 📌 🌍 IP і мережеві налаштування
| Команда                                                       | Для чого використовується | Приклад | Що зробить |
| ------------------------------------------------------------- | ------------------------- | ------- | ---------- |
| `nmcli connection show name`                                  | IP/DNS                    | команда | info       |
| `nmcli connection modify name ipv4.addresses 192.168.1.10/24` | задати IP                 | команда | static IP  |
| `nmcli connection modify name ipv4.gateway 192.168.1.1`       | gateway                   | команда | gateway    |
| `nmcli connection modify name ipv4.dns 8.8.8.8`               | DNS                       | команда | DNS        |
| `nmcli connection modify name ipv4.method manual`             | static                    | команда | manual     |
| `nmcli connection modify name ipv4.method auto`               | DHCP                      | команда | DHCP       |

### 📌 🔄 Перезапуск мережі
| Команда                            | Для чого використовується | Приклад | Що зробить |
| ---------------------------------- | ------------------------- | ------- | ---------- |
| `nmcli networking off`             | вимкнути мережу           | команда | off        |
| `nmcli networking on`              | включити                  | команда | on         |
| `systemctl restart NetworkManager` | перезапуск                | команда | restart    |

### 📌 🔥 Типові сценарії
| Задача                 | Команда                                        |
| ---------------------- | ---------------------------------------------- |
| Подивитись мережі      | `nmcli device wifi list`                       |
| Підключитись до Wi-Fi  | `nmcli device wifi connect SSID password pass` |
| Подивитись підключення | `nmcli connection show`                        |
| Включити інтерфейс     | `nmcli device connect eth0`                    |
| Задати статичний IP    | `nmcli connection modify ...`                  |

### 📌 🧠 Важливо

👉 nmcli vs ip:
| `nmcli`                 | `ip`            |
| ----------------------- | --------------- |
| високий рівень          | низький рівень  |
| зберігає конфіг         | тимчасові зміни |
| працює з NetworkManager | напряму         |

- ip → швидкі/тимчасові зміни
- nmcli → постійна конфігурація 🔥
- ss → порти


📌 🧠 Важливо

👉 ip замінює:
- ifconfig
- route

👉 ss замінює:
- netstat

👉 curl vs wget:
- curl → API, запити
- wget → скачування

## ✅ Підсумок
- мережа → ip
- перевірка → ping
- порти → ss 🔥
- DNS → dig
- HTTP → curl, wget
