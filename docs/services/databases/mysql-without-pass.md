# MySQL без пароля root

Зайти в MySQL без пароля root можна двома основними способами, залежно від налаштувань вашої системи. Найчастіше це робиться через «безпечний режим» зі скиданням пароля.

## Спосіб 1: Через sudo (якщо працює плагін auth_socket) 
У сучасних дистрибутивах Linux (як-от Ubuntu) користувач root в ОС часто може заходити в MySQL без пароля, оскільки система довіряє системному користувачу.
Спробуйте команду: 

```bash
sudo mysql
````

Якщо ви потрапили всередину, ви можете просто встановити новий пароль:
```sql
ALTER USER 'root'@'localhost' IDENTIFIED BY 'ваш_новий_пароль';
```

## Спосіб 2: Через режим --skip-grant-tables (універсальний) 
Якщо перший спосіб не спрацював, потрібно тимчасово вимкнути перевірку паролів. 

Зупиніть службу MySQL:
```bash
sudo systemctl stop mysqld
```

Запустіть MySQL вручну без перевірки прав:
```bash
sudo mysqld_safe --skip-grant-tables &
```

Зайдіть у консоль (пароль тепер не запитають):
```bash
mysql -u root
```

Оновіть пароль:  
Спочатку виконайте `FLUSH PRIVILEGES;`, щоб дозволити редагування користувачів, а потім:
```sql
ALTER USER 'root'@'localhost' IDENTIFIED BY 'новий_пароль';
```

Поверніться у нормальний режим:
- Вийдіть з консолі (exit), зупиніть процес mysqld та запустіть сервіс звичайно:
```bash
sudo systemctl start mysqld
```
 
## Спосіб 3: Використання файлу .my.cnf 
Якщо ви хочете заходити без пароля постійно (наприклад, для скриптів), створіть файл у домашній директорії: 

Файл: `~/.my.cnf`

Вміст:
```ini
[client]
user=root
password="ваш_пароль"
```

> [!NOTE] 
> Важливо:
>  Обов'язково встановіть права `chmod 600 ~/.my.cnf`, щоб інші не могли прочитати пароль. 

Більше подробиць про відновлення можна знайти в інструкціях від [DigitalOcean](https://www.digitalocean.com/community/tutorials/how-to-reset-your-mysql-or-mariadb-root-password) або [Linuxize](https://linuxize.com/post/how-to-reset-a-mysql-root-password/).
