# 🌐 Протоколи HTTP / HTTPS

`Hypertext Transfer Protocol (HTTP)` — це протокол прикладного рівня, який забезпечує передачу гіпертекстових документів у мережах, зокрема в Інтернеті. Він визначає, як клієнти (наприклад, веббраузери) та сервери обмінюються запитами і відповідями, що робить його основою Всесвітньої павутини.

**Ключові факти**  
- **Розробник**: World Wide Web Consortium і Internet Engineering Task Force

- **Початковий реліз**: 1991 р.

- **Поточна основна версія**: HTTP/2 (2015 р.), HTTP/3 на базі QUIC

- **Тип протоколу**: Клієнт-сервер, безстанний

- **Типове використання**: Передача HTML, JSON, зображень, відео

**Історія та розвиток**  
HTTP було створено для обміну документами у системі World Wide Web, започаткованій Тім Бернерс-Лі. Початкові версії (0.9 і 1.0) були простими й обмеженими. HTTP/1.1 (1997 р.) додав підтримку стійких з’єднань і кешування. HTTP/2 представив мультиплексування потоків, а HTTP/3, що базується на протоколі QUIC, покращує затримку та безпеку.

**Принцип роботи**  
HTTP використовує модель «запит-відповідь»: клієнт надсилає запит (GET, POST, PUT тощо), сервер повертає відповідь із кодом стану (наприклад, 200 OK або 404 Not Found). Протокол є безстанним — кожен запит обробляється незалежно, хоча сесійні механізми (cookies, токени) можуть зберігати контекст.

**Безпека та HTTPS**  
Безпечна версія протоколу, HTTPS, додає шифрування через Transport Layer Security (TLS). Це запобігає перехопленню даних та підтверджує автентичність сервера. HTTPS став стандартом для більшості вебсайтів, особливо з конфіденційними даними.

**Значення**  
HTTP залишається фундаментальним протоколом Інтернету. Його гнучкість і розширюваність дозволяють підтримувати сучасні веб-технології, мобільні додатки, API та потокові сервіси, забезпечуючи універсальний спосіб обміну даними між клієнтами і серверами.

## 9. HTTPS (дуже важливо)
📌 що це

👉 HTTP + TLS

`HTTPS` (HyperText Transfer Protocol Secure) — це протокол обміну даними між веббраузером і сервером, який забезпечує шифрування та автентифікацію з’єднання. Він є розширенням стандартного HTTP і використовує криптографічні протоколи для захисту переданої інформації від перехоплення чи змін.

**Ключові факти**
- **Повна назва**: HyperText Transfer Protocol Secure

- **Порт за замовчуванням**: 443

- **Криптографічна основа**: Transport Layer Security / Secure Sockets Layer

- **Стандарт IETF**: RFC 2818 (2000)

- **Основна мета**: Конфіденційність, цілісність, автентичність

**Як працює HTTPS**  
HTTPS додає рівень шифрування між клієнтом і сервером за допомогою протоколів TLS або SSL. При встановленні з’єднання сторони виконують «рукостискання» (handshake), у межах якого узгоджуються криптографічні параметри, перевіряється цифровий сертифікат сервера й створюється симетричний ключ сеансу для подальшого обміну даними.

**Безпека та сертифікати**  
Надійність HTTPS забезпечується через систему цифрових сертифікатів, виданих довіреними центрами сертифікації (CA). Сертифікати підтверджують, що користувач дійсно спілкується з очікуваним сайтом. Рівні перевірки включають DV (Domain Validation), OV (Organization Validation) і EV (Extended Validation).

**Важливість і вплив**  
Використання HTTPS стало стандартом для більшості сайтів завдяки підвищеним вимогам безпеки, конфіденційності й довіри користувачів. Пошукові системи, такі як Google, віддають перевагу HTTPS у рейтингу сторінок, а сучасні браузери позначають HTTP-сайти як «небезпечні».

**Поточний стан**  
Сучасна реалізація HTTPS спирається майже виключно на TLS 1.2 і TLS 1.3, що забезпечують кращу продуктивність і стійкість до атак. Поширення безкоштовних сертифікатів через сервіси на кшталт Let's Encrypt зробило протокол доступним для всіх вебресурсів.
