# 🌍 Протокол DNS (Domain Name System)

`The Domain Name System (DNS)` — це ієрархічна та розподілена система імен, яка перетворює зрозумілі людині доменні імена на числові IP-адреси. Вона є фундаментальним компонентом Інтернету, який дозволяє користувачам підключатися до вебсайтів без необхідності запам’ятовувати числові адреси.

DNS працює подібно до телефонного довідника: вводячи домен (наприклад, example.com), користувач отримує IP-адресу сервера, де розташований сайт, завдяки чому браузер може встановити з’єднання.

**Ключові факти**
- **Запроваджено**: 1983 рік (RFC 882/883; стандартизовано в RFC 1034–1035)

- **Тип системи**: Ієрархічна, розподілена база даних

- **Основні рівні**: Root, TLD, авторитетні сервери

- **Керує**: Internet Corporation for Assigned Names and Numbers через IANA

- **Протокол безпеки**: DNSSEC для автентичності даних

**Ієрархія та компоненти**  
DNS організовано у вигляді дерева доменів. На вершині — кореневі сервери, які знають місцезнаходження серверів доменів верхнього рівня (TLD, наприклад .com, .org, .ua). Далі йдуть авторитетні сервери, що містять записи для конкретних доменів. Запити користувачів обробляють DNS-резолвери, які послідовно звертаються до цих рівнів, доки не отримають IP-адресу потрібного ресурсу.

**Типи запитів і кешування**  
Система підтримує три основні типи запитів: рекурсивні, ітеративні та нерекурсивні. Щоб прискорити роботу, DNS застосовує кешування — тимчасове зберігання результатів пошуку із заданим часом життя (TTL). Це зменшує трафік і прискорює завантаження сторінок при повторних зверненнях.

**Безпека та розширення DNSSEC**  
DNS історично вразлива до атак, як-от підміна кешу (cache poisoning). Протокол DNSSEC (DNS Security Extensions) додає криптографічні підписи до DNS-записів, що дозволяє перевіряти їх достовірність і захищає користувачів від перенаправлення на фальшиві ресурси.

**Значення для Інтернету**  
Без DNS глобальна мережа була б практично непридатною для широкого використання. Ця система забезпечує зручність доступу, стабільність маршрутизації, а також підтримує такі сервіси, як електронна пошта та хмарні ресурси, роблячи DNS одним із найважливіших технологічних фундаментів сучасного Інтернету.

## 1. Що це і навіщо

👉 DNS відповідає на питання:
> “Який IP відповідає цьому доменному імені?”

📌 Приклад
```bash
google.com → 142.250.74.14
```

📌 Чому це потрібно
- людям зручно пам’ятати імена
- комп’ютери працюють з IP

##  2. Архітектура DNS (дуже важливо)
📌 Ієрархія
```bash
Root (.)
 └── TLD (.com, .org)
      └── Domain (google.com)
           └── Subdomain (www.google.com)
```

📌 Типи DNS серверів  
🔹 Root servers
- знають про TLD

🔹 TLD servers
- знають про домени (.com)

🔹 Authoritative servers
- знають точну IP

🔹 Recursive resolver
- виконує запит за тебе

## 3. Як працює DNS (покроково)
📌 процес

1. клієнт → resolver
2. resolver → root
3. root → TLD
4. TLD → authoritative
5. відповідь назад

👉 це називається:
- recursion + iteration

## 4. Структура DNS пакета
```bash
Header
Question
Answer
Authority
Additional
```
🔍 Header
- ID
- flags
- counts

🔍 Question
```bash
google.com IN A
```

🔍 Answer
```bash
google.com 300 IN A 142.250.74.14
```
