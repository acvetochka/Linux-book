# Протокол ICMP (Internet Control Message Protocol)

`Internet Control Message Protocol (ICMP)` — це мережевий протокол управління та діагностики, який є частиною пакету протоколів TCP/IP. Він використовується для обміну службовими повідомленнями між пристроями в мережі, повідомляючи про помилки, статус і доступність мережевих вузлів. Протокол є ключовим для роботи інструментів на кшталт ping і traceroute.

**Основні факти**
- **Тип протоколу**: Мережевий контрольний протокол

- **Рівень моделі OSI**: Мережевий (Layer 3)

- **Використовує транспорт**: Безпосередньо через IP (номер протоколу 1)

- **Рік стандартизації**: 1981 (RFC 792)

- **Основне призначення**: Повідомлення про помилки та перевірка зв’язності

**Функції та структура**  
ICMP передає короткі службові повідомлення, вбудовані у пакети IP. Повідомлення мають тип і код, що визначають їхню функцію, наприклад, «Echo Request» та «Echo Reply» для перевірки доступності вузла або «Destination Unreachable» для індикації проблем з маршрутом. ICMP не використовується для передачі користувацьких даних, а лише для сигналізації.

**Варіанти та розширення**  
Окрім базової версії для `IPv4`, існує `ICMPv6`, розроблений для `IPv6`. ICMPv6 не лише повідомляє про помилки, а й підтримує автоконфігурацію адрес та виявлення сусідів, що робить його важливою частиною механізму роботи IPv6-мереж.

**Безпека та обмеження**  
Хоч ICMP критично важливий для діагностики, він може бути використаний зловмисниками для атак (наприклад, `ping flood` або `smurf attack`). Через це адміністратори часто фільтрують або обмежують ICMP-трафік на міжмережевих екранах, балансуючи між безпекою й доступністю діагностичних засобів.

## 1. Що це і навіщо

👉 ICMP — це службовий протокол для IP
> “повідомляє про стан мережі та помилки доставки”

📌 Важливо
- НЕ передає дані користувача
- використовується для:
  - діагностики
  - повідомлень про помилки

## 2. Де він знаходиться

👉 працює поверх IP:
> IP → ICMP

👉 у полі IP:
```bash
Protocol = 1
```

## 3. СТРУКТУРА ICMP
📌 Формат
```bash
Type
Code
Checksum
Data
```

🔍 Поля

🔹 Type

👉 тип повідомлення

🔹 Code

👉 деталізація

🔹 Checksum

👉 перевірка

🔹 Data

👉 залежить від типу

## 4. ОСНОВНІ ТИПИ ICMP
🔹 1. Echo Request (ping)
```bash
Type 8
```
🔹 2. Echo Reply
```bash
Type 0
```
👉 використовується в:
```bash
ping google.com
```

🔹 3. Destination Unreachable
```bash
Type 3
```

Code приклади:
| Code | Значення            |
| ---- | ------------------- |
| 0    | network unreachable |
| 1    | host unreachable    |
| 3    | port unreachable    |

🔹 4. Time Exceeded
```bash
Type 11
```
👉 використовується в:
```bash
traceroute
```

🔹 5. Redirect
```bash
Type 5
```
👉 змінює маршрут

## 5. ICMP у tcpdump
📌 команда
```bash
tcpdump -I
```

📤
```bash
ICMP echo request, id 1234, seq 1
ICMP echo reply, id 1234, seq 1
```

🔍 Розбір
- `echo request` → ping
- `seq` → номер пакета

## 6. Як працює ping
📌 процес
1. відправляє ICMP Echo Request
2. отримує Echo Reply
3. вимірює RTT
   
📌 команда
```bash
ping 8.8.8.8
```
📤
```bash
64 bytes from 8.8.8.8: icmp_seq=1 ttl=117 time=20 ms
```

🔍 Розбір
- `icmp_seq` — номер
- `ttl` — TTL відповіді
- `time` — затримка

## 7. ICMP і traceroute

👉 traceroute використовує:

- TTL = 1 → ICMP Time Exceeded
- TTL = 2 → наступний hop

## 8. ВАЖЛИВІ НЮАНСИ
❗ ICMP можуть блокувати

👉 firewall часто:

- блокує ping
- але мережа працює

❗ ICMP не гарантує доступність

👉 ping працює ≠ сервіс працює

❗ ICMP важливий для MTU

👉 використовується:
- Path MTU Discovery

## 9. ICMP і помилки
📌 приклад
```bash
Destination Host Unreachable
```
👉 означає:
> немає маршруту

📌 TTL exceeded
```bash
Time to live exceeded
```
👉 пакет “помер”

## 10. ICMP і безпека
❗ може використовуватись для атак:
- ping flood
- network scanning

❗ але блокувати повністю НЕ можна

👉 зламається:

- MTU discovery
- діагностика

## 11. Зв’язок з іншими
| Протокол | Роль         |
| -------- | ------------ |
| IP       | доставка     |
| ICMP     | повідомлення |
| TCP      | дані         |
