# Протокол IP (IPv4)

`Інтернет-протокол` (Internet Protocol, `IP`) — це основний мережевий протокол моделі TCP/IP, який відповідає за адресацію, маршрутизацію та доставку пакетів даних між пристроями в комп’ютерних мережах. Він формує базовий механізм, завдяки якому функціонує інтернет, забезпечуючи глобальну взаємодію систем.

**Ключові факти**
- **Рівень OSI**: мережевий (Layer 3)

- **Основні версії**: IPv4 (1983), IPv6 (2000)

- **Тип протоколу**: безз’єднальний, з найкращими зусиллями (best-effort)

- **Ідентифікація вузлів**: IP-адреси

- **Маршрутизація**: динамічна через таблиці маршрутизації

**Функції та призначення**  
Internet Protocol визначає, як дані інкапсулюються в пакети, яким чином вони адресуються та доставляються до пункту призначення. Протокол не гарантує надійність чи послідовність — ці функції реалізуються на вищих рівнях, наприклад за допомогою TCP. IP забезпечує незалежність мережі від фізичного середовища передачі.

**Версії: IPv4 і IPv6**  
IPv4 використовує 32-бітні адреси, що дозволяє приблизно 4,3 мільярда унікальних адрес. Через вичерпання адрес розроблено IPv6 із 128-бітною адресацією, що забезпечує майже необмежений простір адрес. IPv6 також удосконалює автоконфігурацію, безпеку (IPsec) та маршрутизацію.

**Маршрутизація і фрагментація**  
Пристрої маршрутизаторів аналізують IP-заголовки, щоб визначити найкращий шлях для доставки пакета. Якщо розмір пакета перевищує допустимий для певного сегмента мережі, IP здійснює фрагментацію. У версії IPv6 фрагментація проводиться лише на стороні відправника, спрощуючи роботу маршрутизаторів.

**Значення**  
Internet Protocol є фундаментом глобального інтернету. Він забезпечує універсальний спосіб ідентифікації пристроїв та обміну даними між гетерогенними мережами, що робить його ключовим елементом сучасних комунікаційних систем.

## 1. Що це і навіщо

👉 IP відповідає на питання:

“Як доставити пакет від одного хоста до іншого?”

📌 Головна роль
- адресація (IP)
- маршрутизація
- передача пакетів

⚠️ Важливо

👉 IP:

- ненадійний (no guarantee)
- connectionless
- без підтверджень

👉 надійність = TCP (поверх)

## 2. Що таке IP-пакет
📌 Структура
```bash
[ IP Header ] + [ Data (TCP/UDP/etc) ]
```

## 3. СТРУКТУРА IP HEADER (дуже важливо)
📌 Формат
```bash
Version
IHL
DSCP/ToS
Total Length
Identification
Flags
Fragment Offset
TTL
Protocol
Header Checksum
Source IP
Destination IP
Options (optional)
```

🔍 РОЗБІР КОЖНОГО ПОЛЯ
🔹 Version
- 4 → IPv4

🔹 IHL (Internet Header Length)
- довжина заголовка
- мін: 20 байт

🔹 DSCP / ToS
- пріоритет пакета
- QoS

🔹 Total Length
- весь пакет (header + data)

🔹 Identification
- ID пакета (для фрагментації)

🔹 Flags
| Flag | Значення       |
| ---- | -------------- |
| DF   | Don’t Fragment |
| MF   | More Fragments |

🔹 Fragment Offset
- позиція фрагмента

🔹 TTL (Time To Live)

👉 кількість hop
```bash
TTL = TTL - 1 на кожному роутері
```
👉 якщо 0 → пакет знищується

🔹 Protocol
| Значення | Протокол |
| -------- | -------- |
| 1        | ICMP     |
| 6        | TCP      |
| 17       | UDP      |

🔹 Header Checksum
- перевірка заголовка

🔹 Source IP
- звідки

🔹 Destination IP
- куди

## 6. Фрагментація
📌 коли виникає

👉 якщо пакет > MTU

📌 що відбувається
- пакет ділиться
- кожен має:
  - той самий ID
  - offset

📌 приклад
```bash
Packet 1: offset 0
Packet 2: offset 1480
```

⚠️ проблеми
- фрагменти губляться
- firewall може блокувати
