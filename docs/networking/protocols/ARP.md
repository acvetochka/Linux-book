# Протокол ARP (Address Resolution Protocol)

`Address Resolution Protocol (ARP)` — це мережевий протокол, який використовується для перетворення IP-адреси на MAC-адресу у локальних мережах. Він є важливою частиною стеку TCP/IP, забезпечуючи доставку пакетів між мережевими пристроями в межах однієї фізичної мережі.

**Основні факти**
- **Тип протоколу**: мережевий протокол канального рівня

- **Використовується у**: IPv4 (у IPv6 замінений Neighbor Discovery Protocol)

- **Функція**: відображення IP-адрес на MAC-адреси

- **Стандарт**: Internet Engineering Task Force RFC 826 (1982)

**Як працює**  
ARP використовує широкомовні запити для визначення MAC-адреси пристрою за заданою IP-адресою. Коли комп’ютер потребує доставити пакет іншому пристрою в локальній мережі, він надсилає ARP-запит. Власник відповідної IP-адреси надсилає ARP-відповідь із власною MAC-адресою, після чого дані передаються безпосередньо на канальному рівні.

**Кеш ARP**  
Щоб уникнути повторних запитів, операційні системи зберігають результати ARP у таблиці — кеші ARP. Кожен запис має час життя (TTL), після закінчення якого інформація оновлюється. Цей механізм зменшує навантаження на мережу й прискорює комунікацію.

**Безпека та вразливості**  
ARP не має вбудованих засобів автентифікації, тому піддається атакам типу ARP spoofing або ARP poisoning. Такі атаки дозволяють зловмисникам перенаправляти трафік або здійснювати перехоплення даних. Для захисту застосовують засоби, як-от Dynamic ARP Inspection або статичні ARP-записи.

**Значення**  
ARP залишається фундаментальним протоколом для роботи IPv4-мереж. Його ефективність і простота зробили його ключовим компонентом мережевого зв’язку на канальному рівні, попри відомі ризики безпеки.

## 1. Що це і навіщо

👉 ARP відповідає на питання:
> “Який MAC-адрес має цей IP?”

📌 Проблема, яку він вирішує

Ти знаєш:
```bash
IP: 192.168.1.1
```
Але щоб відправити пакет по Ethernet — потрібно:
```bash
MAC: aa:bb:cc:dd:ee:ff
```
👉 IP ≠ MAC   
👉 ARP = “перекладач”

## 2. Як працює ARP (покроково)
📌 Сценарій

Твій ПК:
```bash
192.168.1.100
```
Хоче відправити пакет:
```bash
192.168.1.1
```

🔹 Крок 1 — перевірка кешу
```bash
ip neigh
```
👉 якщо MAC вже є → OK   
👉 якщо ні → ARP

🔹 Крок 2 — ARP Request (broadcast)
> Who has 192.168.1.1? Tell 192.168.1.100

👉 відправляється:

- на MAC: ff:ff:ff:ff:ff:ff
- ВСІ пристрої отримують

🔹 Крок 3 — ARP Reply

Роутер відповідає:
```bash
192.168.1.1 is at aa:bb:cc:dd:ee:ff
```
👉 вже unicast

🔹 Крок 4 — кешування

👉 запис додається в ARP cache
```bash
ip neigh
```

## 3. ARP таблиця
📌 Команда
```bash
ip neigh
```

📤 Вивід
```bash
192.168.1.1 dev enp0s3 lladdr aa:bb:cc:dd:ee:ff REACHABLE
```

🔬 Розбір
| Поле        | Значення  |
| ----------- | --------- |
| 192.168.1.1 | IP        |
| dev enp0s3  | інтерфейс |
| lladdr      | MAC       |
| REACHABLE   | стан      |

📌 СТАНИ ARP
| Стан       | Значення         |
| ---------- | ---------------- |
| REACHABLE  | працює           |
| STALE      | застарів         |
| DELAY      | перевірка        |
| FAILED     | не знайдено      |
| INCOMPLETE | очікує відповіді |

## 4. Структура ARP пакета
📌 Формат
```bash
Hardware type
Protocol type
Hardware size
Protocol size
Opcode
Sender MAC
Sender IP
Target MAC
Target IP
```
📌 Приклад (логічно)
```bash
Opcode: request
Sender IP: 192.168.1.100
Sender MAC: 08:00:27:xx:xx:xx
Target IP: 192.168.1.1
Target MAC: 00:00:00:00:00:00
```

## 5. ARP у tcpdump
📌 команда
```bash
sudo tcpdump -i enp0s3 arp
```
📤 вивід
```bash
ARP, Request who-has 192.168.1.1 tell 192.168.1.100
ARP, Reply 192.168.1.1 is-at aa:bb:cc:dd:ee:ff
```

🔍 Розбір
- `who-has` → запит
- `is-at` → відповідь


## 6. Керування ARP у Linux
🔹 перегляд
```bash
ip neigh
```

🔹 додати вручну
```bash
ip neigh add 192.168.1.1 lladdr aa:bb:cc:dd:ee:ff dev enp0s3
```

🔹 видалити
```bash
ip neigh del 192.168.1.1 dev enp0s3
```

🔹 очистити кеш
```bash
ip neigh flush all
```

## 7. ВАЖЛИВІ НЮАНСИ
❗ ARP працює тільки в локальній мережі

👉 НЕ проходить через інтернет

❗ якщо IP поза мережею

👉 ARP буде для gateway:
```bash
192.168.1.100 → 8.8.8.8
```
👉 ARP:
```bash
Who has 192.168.1.1?
```

## 8. ARP spoofing (важливо)
📌 атака

зловмисник каже:
> 192.168.1.1 is at MY_MAC

👉 трафік йде через нього

📌 результат
- MITM (man-in-the-middle)

- IP не може працювати в LAN

- IP = ім’я людини
