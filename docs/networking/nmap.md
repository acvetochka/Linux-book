# Nmap — повна довідка

`Nmap (Network Mapper)` — це безплатне програмне забезпечення з відкритим кодом для сканування мереж, створене для виявлення хостів і служб у мережевому середовищі. Його широко застосовують адміністратори систем і фахівці з кібербезпеки для інвентаризації мереж, аудиту безпеки та моніторингу.

**Ключові факти**
- **Розробник**: Gordon Lyon (псевдо Fyodor), Insecure.Org

- **Поточна версія**: 7.98 (2025 р.)

- **Ліцензія**: Nmap Public Source License (на основі GPLv2)

- **Платформи**: Linux, Windows, macOS, *BSD, Solaris

- **Графічний інтерфейс**: Zenmap

**Механізм роботи**  
Nmap аналізує мережевий трафік, використовуючи «сирі» IP-пакети для виявлення активних пристроїв, відкритих портів, служб, операційних систем і версій програмного забезпечення. Інструмент підтримує десятки типів сканування — TCP/UDP-портів, пінг-опитування, визначення ОС (OS fingerprinting) і виявлення брандмауерів.

**Компоненти пакету**  
До складу Nmap входять кілька допоміжних утиліт:

`Zenmap` — офіційна графічна оболонка для зручної роботи з результатами;

`Ncat` — інструмент передачі даних і налагодження мережевих з’єднань;

`Ndiff` — програма для порівняння результатів сканування;

`Nping` — засіб генерації пакетів і аналізу відгуків;

`NSE` (Nmap Scripting Engine) — потужна система сценаріїв для автоматизації перевірок і пошуку вразливостей. 

**Значення та вплив**  
З моменту випуску у 1997 р. Nmap став стандартом у галузі аналізу безпеки. Його часто включають до дистрибутивів Linux, а також використовують у дослідницьких і корпоративних мережах. Програма здобула численні нагороди, зокрема «Security Product of the Year», і навіть з’являлася у популярних фільмах, таких як The Matrix Reloaded та Die Hard 4.

**Поточний стан**  
Проєкт активно розвивається спільнотою через офіційні ресурси nmap.org і репозиторій на GitHub. Оновлення регулярно додають нові скрипти, сигнатури ОС і функції, що робить Nmap актуальним інструментом для аналізу безпеки сучасних мереж.

## 1. Що таке Nmap

Nmap (Network Mapper) — це утиліта для:
- сканування мереж
- виявлення хостів
- визначення відкритих портів
- визначення сервісів і версій
- виявлення ОС
- базового security-аудиту

📦 Використовується для:

- адміністрування мереж
- penetration testing
- аудитів безпеки
- інвентаризації мережі

## 3. Базовий синтаксис
```bash
nmap [OPTIONS] TARGET
```
TARGET може бути:
- IP: 192.168.1.1
- діапазон: 192.168.1.1-100
- мережа: 192.168.1.0/24
- домен: example.com
- список: -iL targets.txt

## 4. Типи сканування (scan techniques)
### 4.1 TCP SYN scan (напіввідкрите)
```bash
nmap -sS 192.168.1.1
```
- найпопулярніше
- швидке
- менш помітне

👉 потребує root

### 4.2 TCP connect scan
```bash
nmap -sT 192.168.1.1
```
- повне TCP з’єднання
- повільніше
- використовується без root

### 4.3 UDP scan
```bash
nmap -sU 192.168.1.1
```
- для UDP сервісів (DNS, SNMP)
- дуже повільне

### 4.4 Stealth / advanced scans
```bash
nmap -sF   # FIN
nmap -sX   # Xmas
nmap -sN   # NULL
```

👉 використовуються для обходу firewall

## 5. Сканування портів
Конкретні порти
```bash
nmap -p 22,80,443 192.168.1.1
```
Діапазон
```bash
nmap -p 1-1000 192.168.1.1
```
Всі порти
```bash
nmap -p- 192.168.1.1
```
Топ порти
```bash
nmap --top-ports 100
```

## 6. Визначення сервісів і версій
```bash
nmap -sV 192.168.1.1
```

Результат:

- порт
- сервіс
- версія

## 7. Визначення ОС
```bash
sudo nmap -O 192.168.1.1
```

використовує TCP/IP fingerprinting

## 8. Агресивний режим
```bash
nmap -A 192.168.1.1
```
Включає:

- OS detection
- version detection
- traceroute
- scripts

⚠️ шумний (видно IDS)

## 9. NSE (Nmap Scripting Engine)
```bash
nmap --script=default 192.168.1.1
```
Приклади:
```bash
nmap --script=vuln 192.168.1.1
nmap --script=http-title 192.168.1.1
nmap --script=ftp-anon 192.168.1.1
```

📂 Категорії:

- auth
- vuln
- discovery
- brute
- exploit

## 10. Виявлення хостів (ping scan)
```bash
nmap -sn 192.168.1.0/24
```
👉 без сканування портів

## 11. Вимкнення ping
```bash
nmap -Pn 192.168.1.1
```
👉 якщо ICMP заблокований

## 12. Швидкість сканування
```bash
nmap -T0  # дуже повільно (stealth)
nmap -T1
nmap -T2
nmap -T3  # default
nmap -T4  # швидко
nmap -T5  # дуже швидко (ризик втрати пакетів)
```

## 13. Вивід результатів
У файл
```bash
nmap -oN output.txt 192.168.1.1
```
XML
```bash
nmap -oX output.xml
```
Grepable
```bash
nmap -oG output.gnmap
```
Все разом
```bash
nmap -oA scan_results
```

## 14. Firewall evasion (обхід)
```bash
nmap -f              # fragmentation
nmap --mtu 8
nmap -D RND:10       # decoys
nmap --source-port 53
```

## 15. Приклади реальних команд
🔍 Швидке сканування мережі
```bash
nmap -T4 -F 192.168.1.0/24
```
🧠 Детальний аудит
```bash
sudo nmap -A -p- 192.168.1.1
```
🔐 Пошук вразливостей
```bash
nmap --script=vuln 192.168.1.1
```
🌐 Тільки активні хости
```bash
nmap -sn 192.168.1.0/24
```

## 16. Стани портів
| Статус                 | Значення              |
| ---------------------- | --------------------- |
| open                   | порт відкритий        |
| closed                 | закритий              |
| filtered               | фільтрується          |
| unfiltered             | доступний, але неясно |
| open       \| filtered | невідомо              |

## 17. Типові помилки

❌ Скани без root → обмежений функціонал  
❌ UDP scan без терпіння  
❌ -T5 в реальній мережі → неточні результати  
❌ скан чужих мереж → незаконно  

## 18. Безпека та законність

👉 Nmap — легальний інструмент, але:

- можна сканувати тільки свої мережі або з дозволом
- інакше це може трактуватись як атака

## 19. Як читати результат

Приклад:
```bash
PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 8.2
80/tcp open  http    Apache 2.4
```
👉 це означає:

- відкриті порти
- сервіс
- версія (для пошуку вразливостей)

## 20. Комбінації (best practice)

💡 Найчастіше використовують:
```bash
sudo nmap -sS -sV -O -T4 target
```
або
```bash
nmap -A target
```

## 21. Коли що використовувати
| Задача        | Команда  |
| ------------- | -------- |
| швидкий огляд | `-F`     |
| повний аудит  | `-A -p-` |
| пошук хостів  | `-sn`    |
| stealth       | `-sS`    |
| без root      | `-sT`    |
