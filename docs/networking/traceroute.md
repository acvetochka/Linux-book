# Утиліта traceroute

`Traceroute` — це програмний інструмент для діагностики мережі, який визначає маршрут і затримку пакетів даних між комп’ютером користувача та віддаленим сервером. Він допомагає виявити, де відбуваються затримки або розриви зв’язку в мережевому з’єднанні.

**Ключові факти**
- **Тип**: мережевий діагностичний інструмент

- **Початковий реліз**: 1987 р.

- **Автор**: Ван Якобсон (Van Jacobson)

- **Основний протокол**: ICMP або UDP

- **Аналог у Windows**: tracert

**Як працює traceroute**  
Traceroute надсилає послідовність пакетів із поступовим збільшенням значення TTL (Time To Live). Кожен маршрутизатор на шляху зменшує TTL на одиницю й надсилає повідомлення ICMP “Time Exceeded”, коли TTL досягає нуля. Вимірюючи час відповіді від кожного вузла, програма відтворює повний маршрут проходження даних.

**Варіації та реалізації**  
Окрім класичного UNIX-інструменту traceroute, існують його модифікації:

- `tracert` — стандартна версія для Windows; використовує ICMP-ехо-запити.

- `mtr` (My Traceroute) — інтерактивна утиліта, що поєднує функції ping і traceroute.

- `pathping` — інструмент Windows із додатковим аналізом втрат пакетів.

**Використання та значення**  
Traceroute є стандартним інструментом для адміністраторів мереж і користувачів, які усувають неполадки підключення. Він допомагає визначити сегменти мережі з високою затримкою, збої або неправильні маршрути. Завдяки прозорості маршруту можна оцінити ефективність маршрутизації в інтернеті або корпоративних мережах.


## Концепція (що відбувається “під капотом”)

traceroute показує шлях пакета від тебе до цілі.

👉 Як це працює:
- використовується поле TTL (Time To Live) в IP-пакеті
- TTL = скільки “стрибків” (hop) пакет може зробити

🔁 Алгоритм
1. Відправляється пакет з TTL=1
2. Перший роутер:
    - зменшує TTL → 0
    - відкидає пакет
    - повертає ICMP "Time Exceeded"
3. Потім TTL=2 → другий роутер
4. І так далі…

👉 В результаті:
>  отримуємо список усіх вузлів (hop)

## 1. Базове використання
🔹 Команда
```bash
traceroute google.com
```
📤 Типовий вивід
```bash
traceroute to google.com (142.250.74.14), 30 hops max, 60 byte packets
 1  192.168.1.1  1.123 ms  0.998 ms  1.002 ms
 2  10.10.0.1    5.234 ms  5.111 ms  5.098 ms
 3  172.16.0.1   10.345 ms  10.221 ms  10.199 ms
 4  * * *
 5  142.250.74.14  25.123 ms  25.001 ms  24.998 ms
```

## 2. РОЗБІР КОЖНОГО РЯДКА
🔹 Заголовок
```bash
traceroute to google.com (142.250.74.14), 30 hops max, 60 byte packets
```
Поля:
- google.com — ціль
- (142.250...) — IP після DNS
- 30 hops max — максимум TTL
- 60 byte packets — розмір пакета

🔹 Рядок хопа
```bash
1 192.168.1.1 1.123 ms 0.998 ms 1.002 ms3`
```

Розбір:  
`1`
- номер hop
  
`192.168.1.1`
- IP вузла (роутер)

`1.123 ms 0.998 ms 1.002 ms`
- 3 запити (за замовчуванням)
- час RTT (round-trip time)

🔹 `* * *`  
`4 * * *`

👉 означає:
> вузол не відповів

Причини:

- firewall блокує ICMP
- rate limiting
- вузол не доступний

🔹 Останній рядок
```bash
5 142.250.74.14 25 ms ...
```
👉 це ціль

## 3. Протоколи traceroute
🔹 За замовчуванням (Linux)
- UDP (порт 33434+)

🔹 Варіанти:  
ICMP
```bash
traceroute -I google.com
```
👉 як `ping`

TCP
```bash
traceroute -T google.com
```
👉 корисно якщо firewall блокує UDP

## 4. ВСІ ВАЖЛИВІ ОПЦІЇ
🔹 -n (без DNS)
```bash
traceroute -n 8.8.8.8
```
📤 швидше (не резолвить імена)

🔹 -m (max hops)
```bash
traceroute -m 5 google.com
```
📤 максимум 5 вузлів

🔹 -q (кількість проб)
```bash
traceroute -q 1 google.com
```
👉 1 запит замість 3

🔹 -w (timeout)
```bash
traceroute -w 1 google.com
```
👉 чекати 1 секунду

🔹 -p (порт)
```bash
traceroute -p 80 google.com
```
👉 змінити порт (важливо для обходу firewall)

🔹 -s (source IP)
```bash
traceroute -s 192.168.1.100 google.com
```
👉 з якого IP відправляти

🔹 -i (інтерфейс)
```bash
traceroute -i enp0s3 google.com
```
👉 через який інтерфейс

🔹 -T (TCP SYN)
```bash
traceroute -T -p 443 google.com
```
👉 імітує HTTPS

🔹 -U (UDP явно)
```bash
traceroute -U google.com
```
🔹 -f (початковий TTL)
```bash
traceroute -f 5 google.com
```
👉 почати з 5 hop

🔹 -F (don’t fragment)
```bash
traceroute -F google.com
```
👉 тест MTU

🔹 --mtu (Linux специфічно)
```bash
traceroute --mtu google.com
```
👉 показує MTU по шляху

## 5. ПРАКТИЧНІ ПРИКЛАДИ
📌 1. Проблема з мережею
```bash
traceroute google.com
```
👉 якщо зупинилось на 2-му hop:

проблема у провайдера

📌 2. Firewall блокує
```bash
traceroute -T -p 443 google.com
```
👉 працює → UDP був заблокований

📌 3. Повільна мережа

дивимось:
- де росте latency

## 6. ОБМЕЖЕННЯ
❗ traceroute НЕ гарантує:
- точний шлях (маршрути можуть змінюватись)
- що всі вузли покажуться

❗ Може бути:
- asymmetric routing
- ICMP blocking
- NAT приховує вузли

## 7. ВАЖЛИВІ ВИСНОВКИ

👉 traceroute показує:

- де затримка
- де packet loss
- де обрив

👉 це ключовий інструмент:

- sysadmin
- DevOps
- мережевий інженер
