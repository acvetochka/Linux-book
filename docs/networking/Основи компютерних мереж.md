# Основи комп'ютерних мереж

## 1. Що таке мережа (простими словами)

`Мережа` — це спосіб, як комп’ютери спілкуються між собою.

👉 Уяви:
- комп’ютер = людина
- IP-адреса = номер телефону
- пакети = повідомлення

Комп’ютери не передають одразу великий файл, а розбивають його на маленькі частини (пакети).

## 2. Як передаються дані (ключова ідея)

Коли ти відкриваєш сайт:

1. Ти вводиш google.com
2. Система знаходить IP
3. Відправляє запит
4. Отримує відповідь

Це виглядає так:
> Ти → роутер → інтернет → сервер → назад до тебе

## 3. Модель TCP/IP (основа всього)

Це головна концепція мереж.

4 рівні:
### 1. Application (рівень програм)
- HTTP (сайти)
- FTP (файли)
- DNS (імена)

### 2. Transport (доставка)

Тут головні протоколи:
- TCP — надійний (перевіряє доставку)
- UDP — швидкий (але без гарантій)

👉 Приклад:
- TCP → завантаження сайту
- UDP → стрімінг, ігри

### 3. Internet (маршрутизація)

Тут працює:

👉 IP address

Він визначає:
- куди відправити пакет
- звідки він прийшов

### 4. Link (фізичний рівень)
- Ethernet
- Wi-Fi
- MAC-адреса

## 4. IP-адреса (дуже важливо)
📌 Що це:

Унікальний ідентифікатор комп’ютера в мережі

Приклад:
> 192.168.1.10

📌 Типи IP  
🔹 Локальні (private)
- 192.168.x.x
- 10.x.x.x
- 172.16.x.x

👉 Використовуються вдома

🔹 Публічні (public)

👉 Видимі в інтернеті

## 5. Маска підмережі (subnet)

Наприклад:
> 192.168.1.10/24

👉 /24 означає:
- перші 24 біти — мережа
- решта — пристрої

Простою мовою:  
👉 визначає “хто у твоїй мережі”

## 6. Gateway (шлюз)

👉 Це вихід в інтернет

Зазвичай:
> 192.168.1.1

📌 Це твій роутер

## 7. Маршрутизація (routing)

Комп’ютер вирішує:

👉 “Куди відправити пакет?”

Є таблиця маршрутів:
```bash
ip route
```
Приклад:
```ini
default via 192.168.1.1
```
👉 все невідоме → через роутер

## 8. DNS (перекладач імен)

👉 Domain Name System

Перетворює:
> google.com → 142.250.x.x

Без DNS ти б вводила тільки IP 😄

## 9. Порти

- IP = будинок
- Порт = квартира

Приклади:
- 80 → HTTP
- 443 → HTTPS
- 22 → SSH

## 10. Як виглядає повний процес

Ти відкриваєш сайт:
1. DNS → знаходить IP
2. TCP → встановлює з'єднання
3. HTTP → запитує сторінку
4. Сервер відповідає

## 11. Локальна мережа (LAN)

У тебе вдома:
- комп’ютер
- телефон
- роутер

Всі в одній мережі:
> 192.168.1.x

## 12. NAT (дуже важливо)

👉 Network Address Translation

Роутер:
- має 1 публічний IP
- роздає приватні IP

👉 Він “перекладає” адреси

## 13. Firewall (брандмауер)

👉 Контролює:
- що входить
- що виходить

## 14. Інтерфейс у Linux

Це “мережевий адаптер”:
- eth0 — кабель
- wlan0 — Wi-Fi
- lo — localhost

## 15. Як це пов’язано з Linux

Тепер команди стають логічними:
- ip addr → твій IP
- ip route → куди йдуть пакети
- ping → чи доходить пакет
- ss → які порти відкриті

## 16. Головна ідея (запам’ятай)

👉 Мережа =
> IP + маршрути + DNS + протоколи

💡 Дуже проста аналогія
| Мережа | Життя               |
| ------ | ------------------- |
| IP     | адреса              |
| DNS    | телефонна книга     |
| пакет  | лист                |
| порт   | квартира            |
| роутер | пошта               |
| TCP    | рекомендований лист |

## Якщо коротко

Щоб працював інтернет, потрібно:
1. IP адреса
2. Gateway
3. DNS
4. Працюючий інтерфейс
