# Налаштування та контроль мережі

## 🌐 1. Мережеві інтерфейси (теорія)
📌 Що таке мережевий інтерфейс

Мережевий інтерфейс — це логічне представлення пристрою, через який система обмінюється даними.

👉 Це НЕ обов’язково фізична карта — це може бути:
- реальна мережева карта
- віртуальний інтерфейс
- тунель
- loopback

📌 Типи інтерфейсів   

🔹 Фізичні
- eth0, enp0s3 — Ethernet
- wlan0 — Wi-Fi

🔹 Віртуальні
- lo — loopback (127.0.0.1)
- docker0 — Docker
- virbr0 — KVM/libvirt
- tun0 — VPN

🔹 Спеціальні
- VLAN (eth0.10)
- bridge (br0)
- bond (агрегація каналів)

📌 Основні параметри інтерфейсу
- IP адреса
- MAC адреса
- MTU
- стан (UP/DOWN)
- RX/TX статистика

## 2. Перегляд стану мережі — команда ip

👉 ip — головна сучасна утиліта

### 2.1 ip addr show
🔹 Команда
```bash
ip addr show
```

🔹 Приклад виводу
```
2: enp0s3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 
    inet 192.168.1.100/24 brd 192.168.1.255 scope global dynamic enp0s3
    inet6 fe80::a00:27ff:fe4e:66a1/64 scope link 
    link/ether 08:00:27:4e:66:a1 brd ff:ff:ff:ff:ff:ff
```

🔍 Детальний розбір  
`2: enp0s3`
- 2 — індекс інтерфейсу
- enp0s3 — ім’я

`<BROADCAST,MULTICAST,UP,LOWER_UP>`
- BROADCAST — підтримка широкомовлення
- MULTICAST — підтримка multicast
- UP — інтерфейс увімкнений
- LOWER_UP — фізичне з’єднання є

`mtu 1500`
- максимальний розмір пакету

`inet 192.168.1.100/24`
- IPv4 адреса
- /24 — маска

`brd 192.168.1.255`
- broadcast адреса

`scope global`
- область дії:
  - global — доступний у мережі
  - link — тільки локально

`dynamic`
- отриманий через DHCP

`inet6 fe80::...`
- IPv6 адреса

`link/ether 08:00:27:4e:66:a1`
- MAC адреса

## 2.2 Опції ip addr
🔹 Тільки один інтерфейс
```bash
ip addr show dev enp0s3
```

🔹 Тільки IPv4
```bash
ip -4 addr
```

🔹 Тільки IPv6
```bash
ip -6 addr
```

🔹 Короткий формат
```bash
ip -br addr
```


📌 Вивід:
```
enp0s3    UP   192.168.1.100/24
```

## 3. Таблиця маршрутизації
📌 Команда
```bash
ip route
```

🔹 Приклад
```bash
default via 192.168.1.1 dev enp0s3
192.168.1.0/24 dev enp0s3 proto kernel scope link src 192.168.1.100
```

🔍 Розбір

`default via 192.168.1.1`
- default маршрут (все інше)
- gateway

`dev enp0s3`
- через який інтерфейс

`proto kernel`
- створено ядром

`scope link`
- локальна мережа

`src 192.168.1.100`
- IP, який буде використано

📌 Опції  

🔹 Детально
```bash
ip route show table main
```
🔹 Для конкретного IP
```bash
ip route get 8.8.8.8
```
📌 покаже:
> через який маршрут піде пакет

## 4. ARP таблиця

👉 Address Resolution Protocol

📌 Команда
```bash
ip neigh
```
🔹 Вивід
```bash
192.168.1.1 dev enp0s3 lladdr aa:bb:cc:dd:ee:ff REACHABLE
```

🔍 Розбір
- IP → MAC відповідність
- lladdr — MAC
- REACHABLE — статус

🔹 Стани:
- REACHABLE
- STALE
- DELAY
- FAILED

📊 5. Статистика інтерфейсів
📌 Команда
```bash
ip -s link
```
🔹 Вивід
```bash
RX: bytes  packets  errors  dropped
TX: bytes  packets  errors  dropped
```

🔍 Розбір

RX
- отримані дані
TX
- відправлені
errors
- помилки
dropped
- втрачені пакети

## 6. Низькорівнева зміна параметрів
📌 Приклади

🔹 Увімкнути інтерфейс
```bash
ip link set enp0s3 up
```
🔹 Змінити MTU
```bash
ip link set enp0s3 mtu 1400.
```
🔹 Додати IP
```bash
ip addr add 192.168.1.200/24 dev enp0s3
```

## 7. Налаштування через конфіги

🟢 Netplan (Ubuntu)
```yml
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: true
```

🟢 systemd-networkd
```yml
[Match]
Name=enp0s3

[Network]
DHCP=yes
```

## 8. Статичні маршрути
📌 Команда
```bash
ip route add 10.0.0.0/24 via 192.168.1.1
```
📌 означає:

- мережа 10.0.0.0
- через gateway

## 9. NetworkManager

👉 NetworkManager

📌 Перегляд
```bash
nmcli device status
```
🔹 Вивід
```
DEVICE   TYPE      STATE      CONNECTION
enp0s3   ethernet  connected  Wired connection
```

🔍 Розбір
- DEVICE — інтерфейс
- TYPE — тип
- STATE — стан
- CONNECTION — профіль

📌 Підключення
```bash
nmcli connection up "Wired connection"
```

## 10. DNS
