# Утиліта ss (Socket Statistics)

`ss` — це командно-рядкова утиліта в системах Linux, призначена для відображення детальної статистики та інформації про сокети. Вона є частиною пакета iproute2 і поступово замінила старішу утиліту netstat, забезпечуючи швидшу роботу та ширші можливості для аналізу мережевих з’єднань.

**Ключові факти**
- **Повна назва**: Socket Statistics

- **Пакет**: iproute2

- **Основна функція**: моніторинг і діагностика мережевих сокетів

- **ОС**: Linux

- **Заміна для**: netstat

**Функціональність і можливості**  
ss використовується для виведення інформації про TCP, UDP, UNIX, RAW та інші типи сокетів. Вона дозволяє фільтрувати з’єднання за станом, портом, IP-адресою або протоколом. Також доступна статистика продуктивності, кількість відправлених і отриманих пакетів та інші діагностичні параметри.

**Переваги над netstat**  
Порівняно з netstat, утиліта ss працює значно швидше, оскільки читає дані безпосередньо з файлової системи /proc, не використовуючи проміжні шари. Вона також надає точнішу інформацію про поточний стан сокетів і підтримує розширені параметри фільтрації для аналізу великої кількості з’єднань.

**Приклади використання**  
- `ss -tuln` — показує всі активні TCP і UDP порти.

- `ss -s` — відображає зведену статистику сокетів.

- `ss -p` — показує процеси, які використовують певні сокети.

**Роль у системному адмініструванні**
ss є важливим інструментом для адміністраторів мереж та серверів, допомагаючи швидко діагностувати проблеми зі з’єднаннями, контролювати трафік і оцінювати навантаження на систему в реальному часі.

## Концепція (що це таке)

`ss` — це утиліта для перегляду:

- мережевих з'єднань
- відкритих портів
- сокетів (TCP, UDP, UNIX)

👉 Замінює:
- netstat (старий, повільний)

📌 Що таке сокет?

👉 сокет =
IP + порт + протокол

Наприклад:
```bash
192.168.1.100:443
```

## 1. Базова команда
🔹
```bash
ss
```
📤 Вивід
```bash
Netid State      Recv-Q Send-Q Local Address:Port   Peer Address:Port
tcp   ESTAB      0      0      192.168.1.100:22     192.168.1.10:52344
```

## 2. РОЗБІР КОЖНОГО ПОЛЯ
🔹 Netid
- тип протоколу:
  - tcp
  - udp
  - unix

🔹 State
- стан з'єднання:
  
  | State      | Значення        |
  | ---------- | --------------- |
  | LISTEN     | очікує          |
  | ESTAB      | встановлено     |
  | SYN-SENT   | відправка SYN   |
  | SYN-RECV   | отримано SYN    |
  | TIME-WAIT  | закриття        |
  | CLOSE-WAIT | очікує закриття |
  | CLOSED     | закрито         |


🔹 Recv-Q
- кількість даних в черзі на отримання

🔹 Send-Q
- дані, що очікують відправки

🔹 Local Address:Port
твій IP і порт
🔹 Peer Address:Port
- віддалений вузол

### 2.1 LISTEN сокети
```bash
ss -l
```
📤
```bash
tcp LISTEN 0 128 0.0.0.0:22 0.0.0.0:*
```
🔬 Розбір
- `0.0.0.0:22` — слухає на всіх інтерфейсах
- `*` — будь-який клієнт

## 3. ОСНОВНІ ОПЦІЇ (з прикладами)
🔹 -t (TCP)
```bash
ss -t
```

🔹 -u (UDP)
```bash
ss -u
```

🔹 -l (LISTEN)
```bash
ss -l
```

🔹 -a (ALL)
```bash
ss -a
```


👉 всі сокети

🔹 -n (без DNS)
```bash
ss -n
```
👉 швидше

🔹 -p (процеси)
```bash
ss -p
```
📤
```bash
tcp ESTAB 0 0 192.168.1.100:22 192.168.1.10:52344 users:(("sshd",pid=1234))
```

🔬 Розбір
- sshd — процес
- pid — процес ID

🔹 -e (extended)
```bash
ss -e
```
📤
```bash
uid:1000 ino:12345 sk:ffff888012345678
```

🔬 Поля
- `uid` — user id
- `ino` — inode сокета
- `sk` — kernel socket pointer

🔹 -m (memory)
```bash
ss -m
```
📤
```bash
skmem:(r0,rb87380,t0,tb16384,f0,w0,o0,bl0)
```
🔬
- `rb` — receive buffer
- `tb` — transmit buffer

🔹 -s (summary)
```bash
ss -s
```
📤
```bash
Total: 100
TCP:   20 (estab 5, listen 2)
UDP:   10
```

🔹 -o (timers)
```bash
ss -o
```
📤
```bash
timer:(keepalive,10sec,0)
```

🔹 -4 / -6
```bash
ss -4
ss -6
```

🔹 -K (kill socket ⚠️)
```bash
ss -K dst 192.168.1.10 dport = :80
```
👉 розриває з'єднання

## 4. ФІЛЬТРАЦІЯ (дуже важливо)
🔹 по порту
```bash
ss -t '( sport = :22 )'
```

🔹 по IP
```bash
ss dst 192.168.1.10
```

🔹 по стану
```bash
ss state established
```

🔹 комбіновано
```bash
ss -t state established '( dport = :443 )'
```

## 5. ПРАКТИЧНІ СЦЕНАРІЇ
📌 Хто слухає порт
```bash
ss -tulpn | grep :80
```

📌 Активні з'єднання
```bash
ss -t state established
```

📌 Скільки з'єднань
```bash
ss -s
```

📌 Високе навантаження
```bash
ss -t state time-wait
```


👉 багато → проблема з TCP tuning

## 6. ВАЖЛИВІ НЮАНСИ
❗ TIME-WAIT
- нормальний стан
- але багато → може бути проблема

❗ LISTEN backlog
- Send-Q в LISTEN = backlog
